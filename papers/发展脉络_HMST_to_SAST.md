# 从 HMST 到 SAST：时频分析谐波感知的演进脉络

HMST 提供了高精度的 IF 估计，但对频率间的物理耦合关系"视而不见"。SAST 在不改动 HMST 核心的前提下，叠加物理图 + GAT 信任分配 + 自适应挤压，补上了这块缺失的拼图。

---

## 一、起点：HMST —— 高精度的 IF 估计，但"看不见"谐波关系

HMST 解决了两个核心问题：

1. **显式高阶 IF 公式**：通过 $N \times N$ 上三角矩阵 + 反向代入法得到 $N$ 阶 IF 的解析解，STFT 操作数从 HSST 的 $O(3N-1)$ 降至 $O(N)$
2. **迭代挤压**：$M$ 次嵌套挤压将能量集中到 IF 脊线上，同时保留信号重建能力

但 HMST 有一个架构层面的盲区：**它对每条频率脊线独立处理**。

水泵水轮机中，BPF = 9×fr、GPF = 20×fr、RSI 与 fr 之间存在滑差耦合——这些由机械结构决定的物理关系，HMST 完全不知道，也不利用。当转速波动导致 fr 漂移时，所有谐波同步漂移；但 HMST 对每个频率独立估计 IF，在噪声下高频分量的估计天然更差（高阶导数放大噪声），导致：(1) 谐波间 IF 比例偏离理论值 → 物理不一致；(2) 高频挤压位置不准 → TFR 模糊。

**HMST 有高精度的 IF 估计能力，但缺少一个机制来利用"频率之间存在物理耦合"这个先验知识。SAST 的所有设计都围绕填补这个空白展开。**

---

## 二、SAST 的三步回应

### 第一步：把物理知识结构化 —— Physical Frequency Structure Graph

HMST 看 TFR 看到的是一堆独立的频率脊线。SAST 换了一个视角：**频率不是孤立的点，而是物理图谱上的节点**。

以水泵水轮机为例，图结构由物理知识预设：

```
节点 (频率实体):
  fr (转频, 5.56 Hz)         ← 所有旋转谐波的源头
  RSI_low (涡带, 1.7~2.8 Hz) ← 分数阶，含滑差
  RSI_turb (湍流, 5.6~11.1 Hz)
  BPF (叶片通过, 50.0 Hz)    ← 9×fr，严格锁定
  2×BPF (100.0 Hz)           ← 2×BPF
  GPF (导叶通过, 111.1 Hz)   ← 20×fr
  3×BPF (150.0 Hz)

边 (物理关系):
  BPF ──INTEGER_HARMONIC── fr       比值 = 9，严格锁定
  GPF ──INTEGER_HARMONIC── fr       比值 = 20
  2×BPF──INTEGER_HARMONIC── BPF     比值 = 2
  GPF ──PHASE_LOCKED────── BPF      同源锁相（都锁在转轴上）
  RSI ──FRACTIONAL_HARMONIC── fr    比值 ≈ 0.43，可漂移
  RSI ──SLIP_COUPLED─────── fr     瞬态时锁定松动
```

**图拓扑由物理知识固化，训练和推理中不变。** 运行时只更新边特征值（从 HMST/TFR 测量）：

```
e_{ij}(t) = [r_ij, σ_r, Corr_E, d_f]

r_ij   = IF_i / IF_j          实测瞬时比值
σ_r    = std(r_ij)            比值稳定性（窗口统计）
Corr_E = 能量包络相关系数       同源锁相的替代指标（不用 Δφ 避免相位卷绕）
d_f    = |f_i - f_j| / max    归一化频率距离
```

HMST 问的是"这个频率的 IF 是多少"。SAST 的图问的是"这些频率之间在物理上应该有什么样的关系，而当前的测量告诉我们这种关系健康吗"。

### 第二步：让网络学习信任策略 —— Edge-Conditioned GAT

有了图之后，核心问题变成：**在当前时刻，哪些物理边是可信的？可信的边如何帮助判断不可信的分量？**

SAST 用 Edge-Conditioned GAT 来回答这个问题。边特征注入注意力计算：

```
α_{ij} = softmax(LeakyReLU(a^T [W_q h_i || W_k h_j || W_e e_{ij}]))
```

边特征进去后，GAT 自动学到：
- `σ_r` 大（比值波动大）→ α 降低 → 这条边当前不可信
- `Corr_E` 高（能量相关性强）→ α 升高 → 同源分量互相信任
- 滑差边的 α 随滑差程度自适应调节

GAT 输出两个东西：

```
h_i^{(L)} ──→ A_ij = α_{ij}^{(L)}    最终层注意力 = "哪些边可信"（兼作诊断信号）
         ──→ σ_i(t)                  per-node 挤压带宽 = "这个分量该挤多窄"
```

**关键设计选择：GAT 不输出 IF 修正量 Δω。** HMST 的 IF 原封不动用于挤压。原因是：如果网络输出 Δω 去"修正"HMST 的 IF，一旦 Δω 错了，能量会被挤到完全错误的频率位置，产生虚假分量——这在故障诊断中是致命的。而如果网络只输出 σ_sq（挤压带宽），最坏的情况是 TFR 模糊（σ 太大），但频率位置不会错。**模糊的 TFR 下游还能处理，错位的 TFR 直接导致误诊。**

### 第三步：信任决定的落地 —— Adaptive Kernel Synchrosqueezing

GAT 输出的 σ_i(t) 需要从 per-node 映射到每个 TF bin：

```
i*(t, η) = argmin_i |η - IF_raw_i(t)|     每个 TF bin 归属到最近的物理节点
σ_sq(t, η) = σ_{i*}(t)                    继承该节点的挤压带宽
```

然后用高斯软核替代 HMST 的硬 δ 函数做挤压：

```
SAST(t, ω) = ∫ V_x(t, η) · K(ω - ω̂_HMST(t, η); σ_sq(t, η)) dη

K(z; σ) = exp(-z²/(2σ²)) / (√(2π)σ)
```

**效果**：
- 整数谐波（BPF ↔ fr，物理关系强，A 高）→ σ 窄（≈1 bin）→ 硬挤压 → 锐利 TFR
- 滑差分量（RSI ↔ fr，瞬态松动，A 低）→ σ 宽（≈10-15 bin）→ 软挤压 → **保留物理展宽**
- 滑差本身是诊断信号（涡带失稳的早期指标），**不应该被挤掉**

---

## 三、两个范式的根本差异

| | HMST | SAST |
|---|---|---|
| **频率关系** | 各做各的，互不知晓 | 物理图显式建模所有关系 |
| **IF 估计** | 高阶 IF，独立估计 | **HMST IF 原样保留，不做修正** |
| **挤压策略** | 硬 δ 函数，固定迭代次数 | 高斯软核，带宽由 GAT 信任度条件化 |
| **自适应能力** | 固定参数 N, M | σ_sq 逐 TF-bin 自适应；迭代门控自适应停止 |
| **可解释性** | 无 | A_ij(t) 可视化 = 物理关系活跃度热力图 |
| **网络角色** | 无 | **信任分配者**，不是 IF 修正者 |
| **错误模式** | IF 估计误差 | σ 错 → TFR 模糊（安全），不会错位 |

---

## 四、演进逻辑的一句话总结

> HMST 给出了高精度的 IF，但不知道频率之间存在物理关系。SAST 画了一张物理频率关系图，让 GAT 看图决定"信哪些关系、挤多窄"，HMST 的 IF 不动——网络不修正物理，只分配信任。
